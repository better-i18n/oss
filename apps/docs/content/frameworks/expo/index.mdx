## Overview

`@better-i18n/expo` is an i18next backend plugin that loads translations from the better-i18n CDN. It plugs into your existing i18next + react-i18next setup with zero native modules — meaning it works in **Expo Go** out of the box.

- **i18next backend plugin** - Drop into any existing i18next setup
- **Offline-first** - Persistent caching via AsyncStorage with stale-while-revalidate
- **Expo Go compatible** - Pure JavaScript, no native modules required
- **Device locale detection** - Auto-detect via expo-localization

<Callout type="info">
  This package is a backend plugin for i18next. It does **not** replace react-i18next — it works alongside it. Your existing `useTranslation()` calls stay exactly the same.
</Callout>

## Quick Start

```ts title="i18n.ts"
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import { BetterI18nBackend, getDeviceLocale } from '@better-i18n/expo';

i18n
  .use(BetterI18nBackend)
  .use(initReactI18next)
  .init({
    backend: {
      project: 'your-org/your-project',
    },
    lng: getDeviceLocale({ fallback: 'en' }),
    fallbackLng: 'en',
    interpolation: { escapeValue: false },
  });

export default i18n;
```

```tsx title="App.tsx"
import './i18n';
import { useTranslation } from 'react-i18next';
import { Text } from 'react-native';

function HomeScreen() {
  const { t } = useTranslation();
  return <Text>{t('welcome')}</Text>;
}
```
