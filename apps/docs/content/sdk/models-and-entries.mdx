Better i18n's headless CMS organizes content into **models** and **entries**. Models define the structure (like a database table), and entries are the individual pieces of content (like rows).

## Content Models

A content model defines the shape of your content. Each model has a slug, display name, kind, and optional custom fields.

```typescript
const models = await client.getModels();
```

**Response:**

```json
[
  {
    "slug": "blog-posts",
    "displayName": "Blog Posts",
    "description": "Company blog articles",
    "kind": "collection",
    "entryCount": 24
  },
  {
    "slug": "changelog",
    "displayName": "Changelog",
    "description": "Product updates and releases",
    "kind": "collection",
    "entryCount": 8
  }
]
```

**Model kinds:**
- `collection` — Multiple entries (blog posts, changelog, FAQ)
- `single` — One entry (about page, homepage content)

## Listing Entries

Fetch entries for a content model with filtering, sorting, and pagination.

```typescript
const result = await client.getEntries("blog-posts", {
  status: "published",
  sort: "publishedAt",
  order: "desc",
  language: "en",
  page: 1,
  limit: 10,
});
```

**Response:**

```json
{
  "items": [
    {
      "slug": "hello-world",
      "title": "Hello World",
      "publishedAt": "2026-01-15T10:00:00Z",
      "customFields": {
        "category": "Engineering",
        "readingTime": "5 min"
      }
    }
  ],
  "total": 24,
  "hasMore": true
}
```

### Filtering by Status

```typescript
// Only published entries
const published = await client.getEntries("blog-posts", { status: "published" });

// Only drafts
const drafts = await client.getEntries("blog-posts", { status: "draft" });

// Archived entries
const archived = await client.getEntries("blog-posts", { status: "archived" });
```

### Sorting

```typescript
// Newest first (default)
await client.getEntries("blog-posts", { sort: "updatedAt", order: "desc" });

// Oldest published first
await client.getEntries("blog-posts", { sort: "publishedAt", order: "asc" });

// By creation date
await client.getEntries("blog-posts", { sort: "createdAt", order: "desc" });
```

### Pagination

```typescript
// Page through results
let page = 1;
let hasMore = true;

while (hasMore) {
  const result = await client.getEntries("blog-posts", {
    page,
    limit: 20,
    status: "published",
  });

  console.log(`Page ${page}: ${result.items.length} entries (${result.total} total)`);
  hasMore = result.hasMore;
  page++;
}
```

### Field Selection

Use `fields` to request only the fields you need. When omitted, all fields are returned.

```typescript
// List with only slug and title (faster, less data)
const { items } = await client.getEntries("blog-posts", {
  fields: ["title"],
});

// Include body for excerpt previews
const { items } = await client.getEntries("blog-posts", {
  fields: ["title", "body", "category"],
});
```

Fields that are not requested are omitted from the response. `slug` and `publishedAt` are always included.

## Single Entry

Fetch a full content entry by slug. This returns all fields including the body content, custom fields, and metadata.

```typescript
const post = await client.getEntry("blog-posts", "hello-world", {
  language: "en",
});
```

**Response:**

```json
{
  "id": "entry-uuid",
  "slug": "hello-world",
  "status": "published",
  "publishedAt": "2026-01-15T10:00:00Z",
  "sourceLanguage": "en",
  "availableLanguages": ["en", "tr", "de"],
  "customFields": {
    "readingTime": "5 min",
    "category": "Engineering"
  },
  "title": "Hello World",
  "body": "## Welcome\n\nWelcome to our blog..."
}
```

The `body` field is always a **Markdown string**. Use a Markdown renderer like `react-markdown` or `marked` to display it.

### Field Selection

Request only the fields you need. Base fields (`id`, `slug`, `status`, `publishedAt`, `sourceLanguage`, `availableLanguages`) are always included.

```typescript
// Only fetch title and body
const post = await client.getEntry("blog-posts", "hello-world", {
  fields: ["title", "body"],
});

// Only fetch specific custom fields
const post = await client.getEntry("blog-posts", "hello-world", {
  fields: ["title", "category", "readingTime"],
});
```

### Language Fallback

When you request a language that doesn't have a translation, the SDK falls back to the source language:

```typescript
// If "ja" translation doesn't exist, returns "en" (source) content
const post = await client.getEntry("blog-posts", "hello-world", {
  language: "ja",
});
// post.title → English title (fallback)
```

Check `availableLanguages` to see which translations exist.

## Custom Fields

Content models can define custom fields beyond the base fields (title, slug, body). Custom field values are returned in the `customFields` object.

```typescript
const post = await client.getEntry("blog-posts", "hello-world");

// Access custom fields
console.log(post.customFields.readingTime); // "5 min"
console.log(post.customFields.category);    // "Engineering"
```

For type-safe custom fields, see the [TypeScript guide](/sdk/typescript).
