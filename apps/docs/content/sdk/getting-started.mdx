## Setup

<Steps>
  <Step>
    ### Get your API Key

    1. Go to [dash.better-i18n.com](https://dash.better-i18n.com)
    2. Navigate to **Settings** → **API Keys**
    3. Select **Content** as the type
    4. Create and copy your API key

    <Callout type="warning">
      Keep your API key secret. Never commit it to version control. Use environment variables instead.
    </Callout>
  </Step>

  <Step>
    ### Create a client

    ```typescript title="lib/content.ts"
    import { createClient } from "@better-i18n/sdk";

    export const content = createClient({
      project: "your-org/your-project",
      apiKey: process.env.BETTER_I18N_API_KEY!,
    });
    ```

    Find your project identifier in the dashboard URL: `dash.better-i18n.com/{org}/{project}`
  </Step>

  <Step>
    ### Fetch content

    ```typescript
    import { content } from "./lib/content";

    // List all content models
    const models = await content.getModels();
    console.log(models);
    // [{ slug: "blog-posts", displayName: "Blog Posts", kind: "collection", entryCount: 12 }]

    // List entries with pagination
    const { items, total, hasMore } = await content.getEntries("blog-posts", {
      status: "published",
      limit: 10,
    });

    // Get a single entry
    const post = await content.getEntry("blog-posts", "hello-world");
    console.log(post.title, post.bodyMarkdown);
    ```
  </Step>
</Steps>

## Environment Variables

```bash title=".env"
BETTER_I18N_API_KEY=bi-your-api-key
BETTER_I18N_PROJECT=acme/web-app
```

| Variable | Required | Description |
| --- | --- | --- |
| `BETTER_I18N_API_KEY` | Yes | Your API key from the dashboard |
| `BETTER_I18N_PROJECT` | Yes | Project identifier in `org/project` format |

<Callout type="tip">
  Your project identifier matches the dashboard URL path: `dash.better-i18n.com/acme/web-app` → `"acme/web-app"`
</Callout>

## Configuration Options

| Option | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| `project` | `string` | Yes | — | Project identifier in `org/project` format |
| `apiKey` | `string` | Yes | — | API key for authentication |
| `apiBase` | `string` | No | `https://dash.better-i18n.com` | Custom API base URL |
| `debug` | `boolean` | No | `false` | Log request URLs and responses to console |

## Debugging

Enable debug mode to see exactly what the SDK is doing:

```typescript
const client = createClient({
  project: "acme/web-app",
  apiKey: process.env.BETTER_I18N_API_KEY!,
  debug: true,
});
```

This logs all requests and responses to the console:

```
[better-i18n] Client initialized { apiBase: "https://dash.better-i18n.com", org: "acme", project: "web-app", base: "https://dash.better-i18n.com/api/v1/content/acme/web-app" }
[better-i18n] → getEntries(blog-posts) https://dash.better-i18n.com/api/v1/content/acme/web-app/models/blog-posts/entries?status=published
[better-i18n] ← 200 OK
[better-i18n]   Response: {"items":[...],"total":24,"hasMore":true}
```

## Framework Integration

The SDK works with any JavaScript framework. For framework-specific wrappers:

- **Next.js** — Use [`@better-i18n/next`](/frameworks/nextjs) for server components with ISR caching
- **React / TanStack Start** — Use [`@better-i18n/use-intl`](/frameworks/tanstack-start) for React hooks
- **Any framework** — Use `@better-i18n/sdk` directly with `fetch()`
