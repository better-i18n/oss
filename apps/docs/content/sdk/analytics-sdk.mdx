The `@better-i18n/content` package is the official client SDK for Better i18n Content Analytics. It ships framework adapters for Next.js, React, Expo, Svelte, Vue, and vanilla JS, with `sendBeacon`-first transport that never blocks your page.

<Callout type="info">
  For the data model, transport details, and privacy model, see [Analytics Overview](/sdk/analytics). For the complete method reference, see [Analytics API Reference](/sdk/analytics-api).
</Callout>

## Configure

You need two values from your Better i18n dashboard:

| Value | Where to get it |
|---|---|
| `projectId` | Project Settings → General |
| `apiKey` | API Keys → Public Key (starts with `bi_pub_`) |

The public key is safe to expose to client-side code — it's write-only and scoped to a single project. See [Why is my API key safe?](/sdk/analytics#why-is-my-api-key-safe).

```bash title=".env.local"
NEXT_PUBLIC_BETTER_I18N_PROJECT_ID=your_project_id
NEXT_PUBLIC_BETTER_I18N_KEY=bi_pub_xxxxx
```

## Basic usage

The SDK exposes two primary APIs:

- **`track(event, properties)`** — Fire an event imperatively. Returns immediately, transport happens in the background.
- **`useTrackView(event, properties)`** — React hook that fires once on mount. The default way to track content page views.

```tsx
import { useTrackView } from '@better-i18n/content/adapters/nextjs'

export default function BlogPost({ post }) {
  useTrackView('content.view', {
    entryId: post.id,
    contentModel: 'blog',
    entrySlug: post.slug,
    language: post.locale,
  })

  return <article>{post.body}</article>
}
```
